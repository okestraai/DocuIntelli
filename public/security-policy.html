Information Security Policy

Last updated: February 19, 2026

1. Purpose & Scope

This Information Security Policy establishes the framework for identifying, mitigating, and monitoring information security risks at DocuIntelli AI. It applies to all employees, contractors, and systems that create, access, process, store, or transmit company and customer data, including production infrastructure, development environments, and third-party integrations.

2. Information Security Governance

DocuIntelli AI maintains a security-first organizational culture with the following governance structure:

  • Security ownership: A designated security lead is responsible for maintaining this policy, coordinating risk assessments, and overseeing incident response.
  • Policy review cadence: This policy is reviewed and updated at least annually, or whenever a material change occurs to infrastructure, services, or applicable regulations.
  • Risk assessment: We conduct periodic risk assessments to identify threats to confidentiality, integrity, and availability of customer data. Findings are documented and tracked through remediation.
  • Security awareness: All personnel with access to production systems receive security training upon onboarding and annually thereafter, covering phishing, credential hygiene, secure development practices, and incident reporting.

3. Identity & Access Management

Access to production assets, sensitive data, and critical systems is governed by the principle of least privilege:

  • Authentication: All internal access to production infrastructure (database dashboards, cloud consoles, CI/CD systems) requires multi-factor authentication (MFA).
  • Unique accounts: Each team member uses individually assigned credentials. Shared accounts are prohibited for production systems.
  • Role-based access: Access to infrastructure dashboards, cloud consoles, and deployment pipelines is granted on a role-based, need-to-know basis.
  • Access reviews: Access rights are reviewed quarterly. Accounts for departed personnel are revoked within 24 hours of separation.
  • Service keys: Backend privileged keys are stored exclusively in environment variables on production servers and are never committed to source control or exposed to client-side code.

4. Consumer Authentication

Consumer-facing authentication for the DocuIntelli AI platform includes:

  • Email & password: Accounts are created with email verification via one-time passcode (OTP). Passwords are hashed using industry-standard algorithms and are never stored in plaintext.
  • Google OAuth: Secured OAuth 2.0 flow with PKCE for additional protection against authorization code interception.
  • Session management: Token-based sessions with automatic refresh. Sessions are invalidated on sign-out.
  • Rate limiting: Authentication endpoints are rate-limited per IP to mitigate brute-force and credential-stuffing attacks.
  • Device management: Paid plans enforce per-account device limits with automatic soft-blocking of excess devices.

5. Infrastructure & Network Security

Our production infrastructure is designed with defense-in-depth principles:

  • Encryption in transit: All client-server communication is encrypted using TLS 1.2 or TLS 1.3 exclusively. Older protocols are disabled. Forward-secrecy cipher suites are enforced.
  • Encryption at rest: All data stored in our database is encrypted at rest using AES-256. File storage is encrypted at rest at the infrastructure level.
  • HSTS: HTTP Strict Transport Security is enforced with a long max-age, includeSubDomains, and preload directives.
  • Security headers: Industry-standard security headers (X-Frame-Options, X-Content-Type-Options, Referrer-Policy, and others) are applied to all responses.
  • Network architecture: Production services are accessed exclusively through a secure tunnel. No ports are directly exposed to the public internet. Internal services communicate within an isolated private network.
  • AI infrastructure: Document processing via our AI systems is protected by authenticated access controls. Document content is not retained by the AI system beyond the duration of the processing request.
  • CORS: Whitelist-based Cross-Origin Resource Sharing restricts API access to authorized domains only.
  • Rate limiting: Per-IP rate limits are enforced on all API endpoints, with stricter limits on authentication and upload endpoints to prevent abuse.

6. Data Protection

Customer data is protected at every layer of the application:

  • Row-Level Security: Database-level security policies are enforced on all tables, ensuring users can only access their own documents, subscriptions, and settings — even at the query level.
  • Client/server key separation: The frontend uses restricted API keys subject to row-level security. The backend uses privileged keys only for administrative operations, and these keys are never exposed to client-side code.
  • Signed URLs: Document file access is controlled through time-limited, cryptographically signed URLs that expire after use.
  • Payment data: All payment processing is handled by Stripe under PCI-DSS Level 1 compliance. DocuIntelli AI never stores, processes, or has access to full credit card numbers.
  • No document content training: Customer document content is never used for training, fine-tuning, or improving AI models.

7. Secure Development Practices

  • Version control: All code changes are tracked in Git with branch-based development workflows.
  • Code review: Changes to production code are reviewed before merging.
  • Dependency management: Third-party dependencies are monitored for known vulnerabilities using automated scanning and alerting tools.
  • Secrets management: Secrets and API keys are stored in environment variables, never in source code. Build and deployment configurations prevent accidental inclusion of sensitive files.
  • Input validation: User input is validated and sanitized at API boundaries. File uploads are restricted by type and size.

8. Incident Response

DocuIntelli AI maintains an incident response procedure to detect, contain, and recover from security events:

  • Detection: Automated monitoring via health check endpoints, Cloudflare analytics, and application-level logging identifies anomalies and outages.
  • Containment: Upon detection of a security incident, affected systems are isolated. Compromised credentials are immediately rotated.
  • Notification: Affected users are notified within 72 hours of a confirmed data breach, in compliance with applicable privacy regulations (GDPR, state-level breach notification laws).
  • Post-incident review: Every security incident is followed by a root-cause analysis and remediation to prevent recurrence.

9. Third-Party Risk Management

We carefully evaluate the security posture of all third-party services integrated with our platform:

  • Supabase: SOC 2 Type II certified. Provides database hosting, authentication, and storage with built-in encryption and RLS.
  • Stripe: PCI-DSS Level 1 certified. Handles all payment data processing.
  • Cloudflare: SOC 2, ISO 27001 certified. Provides CDN, DDoS protection, SSL termination, and Zero Trust access to AI infrastructure.
  • Mailjet: GDPR-compliant email delivery service for transactional notifications.

Only the minimum data necessary is shared with each third-party service to fulfill its function. Data sharing agreements and processing terms are reviewed before integration.

10. Business Continuity

  • Data backups: Database backups are managed by our hosting provider with point-in-time recovery. Cache services use persistent storage for durability.
  • Redundancy: Global CDN caching and DDoS protection ensure availability even under adverse network conditions.
  • Containerized deployment: Production services run in isolated containers, enabling rapid recovery and consistent environments across restarts.

11. Policy Compliance & Enforcement

Adherence to this policy is mandatory for all personnel with access to DocuIntelli AI systems. Violations may result in disciplinary action, up to and including termination of access and employment. Questions or concerns regarding this policy should be directed to security@docuintelli.com.

12. Contact

For security-related questions, vulnerability reports, or incident disclosures, contact us at security@docuintelli.com.
